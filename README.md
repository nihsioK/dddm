# DDDM Assignment 2

## Что это за проект

Это проект для второго assignment по `Data-Driven Decision Project`.

Идея проекта: у нас есть бизнес-идея `Staj.kz` — платформа стажировок и early-career вакансий для студентов и выпускников Казахстана.

Чтобы понять, стоит ли такую платформу запускать и в какой сегмент рынка лучше заходить, мы собираем данные о вакансиях с реальных площадок, очищаем их, анализируем и на основе этого делаем выводы.

## Объяснение проекта простыми словами

Для начала у нас есть главный скрипт: `assignment2_pipeline.py`.

Он делает почти всю работу автоматически.

Сначала он идёт в два источника вакансий:

- `hh.kz`
- `Enbek.kz`

Потом он ищет не только слово `стажировка`, а более широкий набор запросов, чтобы собрать не 20-30 вакансий, а нормальный объём рынка.

Используются такие фильтры и запросы:

- `стажировка`
- `стажер`
- `intern`
- `internship`
- `trainee`
- `junior`
- `graduate`
- `студент`
- `без опыта`
- `первая работа`
- `начинающий специалист`

То есть логика такая:
мы собираем не только буквальные стажировки, а весь `early-career` рынок — вакансии для людей без опыта, junior-позиции, trainee/graduate роли и первую работу.

## Что делает скрипт по шагам

### 1. Сбор данных с `hh.kz`

Скрипт ходит в API `hh.kz` и по каждому запросу проходит несколько страниц.

Из каждой вакансии он вытаскивает:

- название вакансии
- компанию
- город
- дату публикации
- зарплату
- опыт
- тип занятости
- график работы
- ссылку
- и дополнительные признаки для анализа

### 2. Сбор данных с `Enbek.kz`

У `Enbek.kz` нет такого удобного открытого API, поэтому там скрипт работает через HTML-страницы:

- сначала читает страницы поиска вакансий
- потом для части вакансий открывает карточку вакансии
- добирает детали: образование, навыки, регион, зарплату, опыт и другие поля

### 3. Приведение к одной структуре

После сбора из двух сайтов данные объединяются в одну общую таблицу.

Это нужно, чтобы вакансии из `hh.kz` и `Enbek.kz` можно было сравнивать между собой.

## Как происходит очистка данных

После сбора скрипт не использует данные как есть, а очищает их.

### Удаление дублей

Сначала удаляются явные дубли:

- одинаковый источник
- одинаковый ID вакансии

Потом удаляются мягкие дубли:

- одинаковое название вакансии
- одинаковая компания
- одинаковый город
- одинаковая дата публикации

### Проверка свежести

Мы оставляем только свежие вакансии.

Сейчас используется окно в `90 дней`.

### Нормализация полей

Скрипт приводит к единому виду:

- города
- графики работы
- типы занятости
- валюты
- даты

### Работа с зарплатой

Если у вакансии есть диапазон зарплаты, скрипт считает усреднённое значение.

Потом он старается привести зарплату к `KZT`, чтобы разные вакансии можно было честно сравнивать.

### Проверка выбросов

Числовые данные по зарплатам проверяются методом `1.5xIQR`.

Если значение выглядит как слишком сильный выброс, оно не используется как нормальная рыночная зарплата.

## Что считается после очистки

Когда данные уже чистые, скрипт считает аналитику:

- сколько вакансий пришло из каждого источника
- в каких городах их больше всего
- какие категории ролей доминируют
- какая доля вакансий публикуется с зарплатой
- какой ценовой сегмент наиболее перспективен

Также рынок делится на сегменты:

- низкий
- средний
- высокий
- премиальный

Сегменты задаются не вручную, а по квартилям реального распределения зарплат.

## Какие файлы появляются на выходе

### Итоговые таблицы

- `data/processed/assignment2_raw_combined.csv` — сырые объединённые данные
- `data/processed/assignment2_cleaned_market_intelligence.csv` — итоговый очищенный датасет
- `data/processed/assignment2_quality_log.csv` — журнал очистки
- `data/processed/assignment2_summary.json` — краткая итоговая сводка

### Графики

Папка:

- `output/figures/`

Там лежат графики по рынку:

- распределение вакансий по источникам
- топ городов
- категории ролей
- ценовые сегменты
- динамика публикаций
- прозрачность зарплат

### Финальный отчёт

- `output/doc/assignment2_report.tex`

Это финальный LaTeX-отчёт для сдачи.

## Как это связано с бизнес-идеей `Staj.kz`

Смысл не в том, чтобы просто собрать вакансии.

Смысл в том, чтобы на основе данных ответить на бизнес-вопросы:

- есть ли в Казахстане достаточный рынок early-career вакансий
- в каких городах лучше запускаться сначала
- какие роли и сегменты встречаются чаще всего
- насколько конкуренты прозрачны по зарплатам
- где у `Staj.kz` есть шанс занять свою нишу

Итоговая идея такая:

- большие платформы покрывают рынок широко
- но они не специализируются именно на студентах
- значит `Staj.kz` может зайти как нишевой продукт для студентов и выпускников без опыта

## Как запустить

Установка зависимостей:

```bash
python3 -m venv .venv
.venv/bin/pip install -e .
```

Запуск пайплайна:

```bash
.venv/bin/python assignment2_pipeline.py
```

## Коротко в одной фразе

Этот проект автоматически собирает вакансии с `hh.kz` и `Enbek.kz`, очищает их, анализирует рынок early-career вакансий Казахстана и готовит данные и выводы для оценки запуска `Staj.kz`.
